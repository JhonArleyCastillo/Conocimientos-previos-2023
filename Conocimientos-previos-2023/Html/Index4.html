    • git init

      Sirve para inicializar el repositorio en la carpeta de destino o servidor, crea un control de versiones un .git.

      git version git -v

      Arroja la version actual de Git.

      git add .

      Agrega el registro de los cambios al repositorio.

      git commit -m

      Permite realizar un comentario a los cambios realizados en el repositorio.

      git push

      Sube los cambios realizados en el repositorio local al repositorio remoto.

      git pull

      Baja los cambios realizados en el repositorio remoto al repositorio local.

      git clone + urlgithub

      Clona un repositorio remoto al repositorio local.

      git status

      Permite ver el estado actual de los archivos.

      git config --global user.name

      Configura el usuario global en el entorno de Git.

      git config --global user.email

      Configura el email del usuario global en el entorno de git.

      git config user.name

      Permite visualizar el usuario global de git en el entorno local.

      git config user.email

      Permite visualizar el email del usuario global de git en el entorno local.

      git branch

      Visualiza la rama en la cual se encuentra el usuario.

      git checkout -b "Nombre de la rama"

      Permite crear una nueva rama llamada "Nombre de la rama".

      git branch -d "Nombre de la rama".

      Borra la rama "Nombre de la rama".

      git checkout "Main"

      Permite ir a la rama Main".

      git merge "rama"

      Fusiona una rama con otra rama en la cual estas activo.

      git tag 0000 Codigo hexagesimal del commit

      Coloca un marcador o etiqueta "0000" en el comentario

      git log

      Visualizas los codigos hexagesimales de los commit realizados.
